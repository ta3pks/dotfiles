  <action>
    Create README.md with comprehensive security section:
    ```markdown
    # Dotfiles Repository
    
    Personal Linux development environment configuration.
    
    ## Quick Start
    
    ```bash
    # Clone the repository
    git clone https://github.com/nikos/dotfiles.git ~/.dotfiles
    cd ~/.dotfiles
    ```
    
    ## Secrets Management
    
    This repository is designed to keep secrets out of version control.
    
    ### Environment Variables
    
    1. Copy the example file:
       ```bash
       cp .env.example .env
       ```
    
    2. Edit `.env` with your actual values:
       ```bash
       $EDITOR .env
       ```
    
    3. Source in your shell configuration:
       ```bash
       # Add to ~/.zshrc or ~/.bashrc
       [ -f ~/.env ] && export $(grep -v '^#' ~/.env | xargs)
       ```
    
    ### What We Don't Commit
    
    The following patterns are blocked by `.gitignore`:
    - API keys and tokens (`*_API_KEY`, `*_TOKEN`)
    - Environment files (`.env`, `.env.*`)
    - SSH private keys (`id_rsa*`, `*.pem`)
    - Credential files (`credentials.json`, `secrets.yaml`)
    - Shell history files (may contain secrets)
    
    ### Pre-Commit Protection
    
    We recommend using gitleaks as a pre-commit hook:
    
    ```bash
    # Install the hook
    gitleaks git --pre-commit
    
    # Or run manually before commits
    gitleaks git --staged --verbose
    ```
    
    ### Secret Scanning
    
    To verify no secrets are committed:
    ```bash
    gitleaks git --verbose
    ```
    
    ## Repository Structure
    
    Each tool has its own directory with configuration files that can be symlinked to the appropriate XDG locations.
    
    ## License
    
    Personal configuration - use at your own discretion.
    ```
