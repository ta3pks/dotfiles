- Exit with the same code as a real run would (0 if all would succeed)
- Dry-run does NOT write to .bootstrap-manifest

<task type="auto">
  <name>Implement --undo with manifest-based restoration</name>
  <files>bootstrap.sh</files>
  <action>
Implement the `--undo` flag handler in bootstrap.sh:

**Undo logic:**
1. Check `.bootstrap-manifest` exists — if not, error "No manifest found. Nothing to undo." and exit 1
2. Read manifest line by line (newest entries first — `tac` the file)
3. For each entry `<target_path>|<source_path>`:
   a. Check if target_path is a symlink pointing into the dotfiles repo
   b. If yes: remove the symlink (`rm`)
   c. Check if `<target_path>.bak` exists
   d. If yes: restore it (`mv <target_path>.bak <target_path>`)
   e. Log each action: "Removed symlink: <path>", "Restored backup: <path>"
4. After processing all entries, remove the `.bootstrap-manifest` file itself
5. Print summary: X symlinks removed, Y backups restored

**Edge cases:**
- Symlink in manifest no longer exists → skip with warning
- Symlink points somewhere else (manually changed) → skip with warning, don't remove
- `.bak` file doesn't exist → just remove symlink, no restore
- Empty manifest → print "Nothing to undo"

**Dry-run interaction:**
- `--undo --dry-run` should preview what would be undone without making changes
  </action>
  <verify>
    <automated>grep -q "\-\-undo" bootstrap.sh && grep -q "Removed symlink\|Restored backup\|Nothing to undo" bootstrap.sh && echo "undo implementation verified"</automated>
  </verify>
  <done>--undo reads .bootstrap-manifest, removes symlinks, restores .bak files, and cleans up manifest</done>
</task>

<task type="auto">
  <name>Harden idempotency edge cases</name>
  <files>bootstrap.sh</files>
  <action>
Review and harden the symlink creation loop for idempotency:

**Edge cases to handle:**
1. **Correct symlink exists** → skip, log "already linked", do NOT append to manifest again
2. **Symlink exists pointing to wrong target** → treat as conflict (apply strategy)
3. **Broken symlink at target** (dangling) → remove it, create correct symlink
4. **Source file doesn't exist in repo** (e.g., deleted tool dir) → warn "source not found: <path>", skip
5. **Target parent directory missing** → `mkdir -p` before linking
6. **Re-run after partial failure** → manifest may have some entries; correct links skipped, failed ones retried

**Verification pass (post-install):**
After all symlinks are created, iterate all entries in manifest and verify:
- Each target is a symlink
- Each symlink resolves to the expected source
- Print verification result: "Verified: X/Y symlinks correct"
- If any fail verification, exit 1

Ensure the script produces IDENTICAL filesystem state whether run once or ten times.
  </action>
  <verify>
    <automated>grep -q "already linked" bootstrap.sh && grep -q "Verified\|verified" bootstrap.sh && grep -q "source not found\|dangling\|broken" bootstrap.sh && echo "idempotency hardening verified"</automated>
  </verify>
  <done>bootstrap.sh handles all idempotency edge cases and verifies symlinks after creation</done>
