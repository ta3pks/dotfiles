    ```markdown
    ## Secrets Management
    
    This repository follows a strict no-secrets-in-version-control policy.
    
    ### Strategy
    
    - **Environment Variables**: All secrets are managed via environment variables
    - **External Management**: Secrets are stored in password managers or system keychains
    - **Template Files**: Use `.env.example` files as templates for required variables
    
    ### Usage
    
    1. Copy `.env.example` to `.env` (if applicable)
    2. Fill in your actual values
    3. Never commit `.env` files (covered by .gitignore)
    
    ### Verification
    
    Run `gitleaks git --verbose .` to scan for accidentally committed secrets.
    
    ### What's Protected
    
    The .gitignore covers:
    - API keys and tokens (*_API_KEY, *_TOKEN)
    - Passwords (*_PASSWORD, *_SECRET)
    - Environment files (.env, .env.*)
    - SSH private keys (.ssh/id_rsa*)
    - AI tool configs (.config/opencode/, .aider*)
    - Shell history files
    ```
    
    Insert after any existing setup/installation section, or at an appropriate location in the README structure.
  </action>
