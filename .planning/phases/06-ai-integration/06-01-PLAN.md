# Plan 06-01: Core Service Layer + MCP Server

**Goal:** Create unified memory service and MCP server for OpenCode integration

## Context

Phase 6 exposes the memory system to AI coding tools. This plan creates:
1. A unified `MemoryService` class that all interfaces will use
2. An MCP server that exposes memory tools to OpenCode

The service layer extracts common logic from the CLI into a reusable module.

## Tasks

<title>Create core service types</title>

<action>
Create `memory/src/core/types.ts` with service interfaces:

```typescript
export interface StoreOptions {
  tags?: string[];
  project?: string | null;
  type?: "pattern" | "decision" | "context" | "preference" | "general";
}

export interface SearchOptions {
  limit?: number;
  project?: string;
  tags?: string[];
}

export interface ListOptions {
  project?: string;
  tag?: string;
  limit?: number;
  offset?: number;
}

export interface ContextOptions {
  maxTokens?: number;
  project?: string;
}
```

<task id="4" type="implementation">
<title>Create MCP server</title>

<action>
Create `memory/src/mcp/server.ts`:

1. Import `@modelcontextprotocol/sdk`
2. Create McpServer instance
3. Register tools:
   - `memory_store` - Store new memory
   - `memory_search` - Semantic search
   - `memory_get` - Get by ID
   - `memory_list` - List with filters
   - `memory_delete` - Delete memory
   - `memory_context` - Get context for query

   - `memory://projects` - List all projects

5. Use MemoryService for all operations
