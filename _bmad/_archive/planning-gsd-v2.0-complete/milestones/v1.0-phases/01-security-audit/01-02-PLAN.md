# Plan 02: Update .gitignore with Secret Patterns

<objective>
Add comprehensive secret patterns to .gitignore to prevent future credential exposure.
</objective>

<context>
@.planning/phases/01-security-audit/01-RESEARCH.md
@.gitignore
</context>

<tasks>

<task type="auto">
  <name>Add secret patterns to .gitignore</name>
  <files>.gitignore</files>
  <action>
    Add comprehensive secret patterns to .gitignore. Append the following section:
    
    ```gitignore
    # ========================================
    # Security: Secret Patterns (Phase 1)
    # ========================================
    
    # API Keys and Tokens
    *_API_KEY
    *_TOKEN
    *_PASSWORD
    *_SECRET
    *_PRIVATE_KEY
    
    # Environment files
    .env
    .env.*
    *.env
    
    # AI tool configs (may contain API keys)
    opencode/.config/opencode/*.json
    .aider*
    
    # Shell history (may contain secrets in commands)
    .bash_history
    .zsh_history
    .local/share/history/
    
    # SSH private keys
    .ssh/id_rsa*
    .ssh/*.pem
    id_ed25519
    id_ecdsa
    
    # Credential files
    credentials.json
    secrets.yaml
    secrets.yml
    *.pem
    *.key
    
    # Browser data
    *Brave*
    *Keybase*
    ```
  </action>
  <verify>Patterns added to .gitignore</verify>
  <done>Comprehensive secret patterns in .gitignore</done>
</task>

<task type="auto">
  <name>Verify patterns are effective</name>
  <files>.gitignore</files>
  <action>
    Create test files to verify gitignore patterns work:
    ```bash
    # Create test files (will be ignored)
    touch test_API_KEY
    touch test_TOKEN
    touch .env.test
    
    # Verify they're ignored
    git status --porcelain | grep -E "(test_API_KEY|test_TOKEN|\.env\.test)" || echo "Patterns working correctly"
    
    # Clean up test files
    rm -f test_API_KEY test_TOKEN .env.test
    ```
  </action>
  <verify>Test files are properly ignored by git</verify>
  <done>Gitignore patterns verified working</done>
</task>

<task type="auto">
  <name>Run gitleaks on current state</name>
  <files></files>
  <action>
    Verify no new secrets can be committed:
    ```bash
    gitleaks dir --verbose . 2>&1 || true
    ```
    Document result.
  </action>
  <verify>Gitleaks directory scan completes</verify>
  <done>Directory scan verification complete</done>
</task>

</tasks>

<verification>
- .gitignore contains comprehensive secret patterns
- Patterns cover: API keys, tokens, env files, SSH keys, credential files
- Test verification confirms patterns work
- Gitleaks directory scan clean
</verification>

<success_criteria>
- [ ] Secret patterns added to .gitignore
- [ ] Patterns tested and working
- [ ] No secrets in current directory state
</success_criteria>

<output>
After completion, create SUMMARY.md with:
- Patterns added to .gitignore
- Verification results
- SEC-02 requirement status
</output>
