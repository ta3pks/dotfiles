<task type="auto">
  <name>Create bootstrap.sh core script</name>
  <files>bootstrap.sh</files>
  <action>
Create `bootstrap.sh` at repo root. Make it executable (`chmod +x`).

**Script structure:**

1. **Shebang and strict mode**: `#!/usr/bin/env bash` with `set -euo pipefail`

2. **Constants**: `SCRIPT_DIR` (resolved via readlink), `MANIFEST_FILE=".bootstrap-manifest"`, `PREREQ_FILE="prerequisites.json"`

3. **Flag parsing**: Parse CLI arguments:
   - `--dry-run`: Set DRY_RUN=1
   - `--undo`: Set UNDO=1 (stub — implemented in Plan 02)
   - `--all`: Set MODE=all
   - `--essential`: Set MODE=essential
   - `--conflict=overwrite|backup|skip`: Set CONFLICT_STRATEGY
   - `--help`: Print usage and exit 0

4. **Core prerequisite check**: Verify `jq` and `whiptail` from prerequisites.json core section. If missing, print error listing all missing and exit 2.

5. **Tool selection**:
   - If `--all`: select all tools from prerequisites.json
   - If `--essential`: select only tools with `category: "essential"`
   - Otherwise: show whiptail checklist
     - Essential tools ON by default, optional tools OFF
     - Priority order: essential first, then optional alphabetically
     - Checklist format: `<tool> <description> <ON|OFF>`

6. **Conflict strategy prompt**:
   - If `--conflict=` flag set, use that
   - Otherwise: whiptail radiolist with Overwrite/Backup/Skip options
   - Default selection: Backup

7. **Per-tool prerequisite check**: For each selected tool, run its `check` command. Collect all failures. If any fail, print all missing and exit 1.

8. **Symlink creation loop**: For each selected tool, iterate its mappings:
   - Expand `~` to `$HOME`
   - For glob patterns (src ending in `*`): expand and create individual symlinks
   - Check target state:
     - Symlink pointing to correct target → skip, log "already linked"
     - Symlink pointing to wrong target → treat as conflict
     - Regular file/directory exists → apply conflict strategy
     - Nothing exists → create symlink
   - Conflict handling:
     - `overwrite`: create `.bak` backup first, then symlink
     - `backup`: create `.bak` backup, then symlink
     - `skip`: log "skipped", continue
   - Create parent dirs with `mkdir -p`
   - Use `ln -sf` for files, `ln -snf` for directories

9. **Summary output**: Print counts of linked, skipped, backed up, errored items.

10. **Verification**: For each created symlink, verify it resolves to the correct target. Report any failures.

**Exit codes**: 0 success, 1 general error, 2 missing core prerequisites.

**Logging**: Use helper functions `log_info`, `log_warn`, `log_error` with colored output (green/yellow/red). Respect `NO_COLOR` env var.
  </action>
